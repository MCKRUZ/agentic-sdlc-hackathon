Module 02 — Out-of-the-Box Usage
Prompt strategy

Three Prompt Types That Always Work

1
Navigation
"Where is X?"
"What are the main entry points for this application?"
Why it works
Grounds the agent in your actual file structure before it does anything.
2
Explanation
"What does X do?"
"Pick the most complex function in this project and explain what it does."
Why it works
Tests the agent's comprehension before trusting its edits.
3
Targeted Change
"Change X to Y"
"Add a JSDoc comment block to the createOrder() function explaining its parameters."
Why it works
Small, reviewable, low-risk — the right kind of first change.
Start with navigation.  →  Then explanation.  →  Then change.   Build trust incrementally.
