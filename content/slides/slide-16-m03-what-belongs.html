Module 03 — Project Context Files

Write It Like You're Onboarding a Contractor

Put This In
  • ✓Tech stack with versions
  • ✓Naming and file organization conventions
  • ✓Error handling patterns
  • ✓Things the agent must NEVER do
  • ✓Links to architectural decision records
  • ✓Current focus area
Leave This Out
  • ✗Secrets or API keys
  • ✗Long prose explanations
  • ✗Generic advice that applies to any project
  • ✗Instructions the agent already follows by default
  • ✗Your full README
The contractor analogy: technically skilled, knows nothing about YOUR project specifically. Write for that person.
