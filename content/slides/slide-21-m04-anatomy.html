# commit-message

Generate a conventional commit message for the staged changes.

## When to Use
After `git add` — before `git commit`.

## Instructions
1. Run `git diff --staged` to see staged changes.
2. Identify the type: feat, fix, refactor, docs, test, chore.
3. Write a commit message: `type: imperative description` (under 72 chars).
4. Do not run `git commit` — print the message only for review.

## Output Format
One line:
`type: description`
Optionally a blank line and a body paragraph explaining WHY.
Opening line
One sentence: what it does. This becomes the description when listing available skills.
When to Use
Context for triggering. Helps you remember when the skill is relevant, especially weeks later.
Instructions
Step-by-step for the agent. Numbered steps reduce ambiguity and keep output consistent.
Output Format
Exact output shape. Define this to avoid the agent deciding format on its own every time.
