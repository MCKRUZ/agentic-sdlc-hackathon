Hands-On Lab
LAB 03

Build a
Skill

35 minutes  ·  You'll leave with a working
skill committed to your repo
Pick something you do repeatedly.
One excellent skill beats six mediocre ones.
What You'll Do
  • Create the skills directory: mkdir -p .claude/commands
  • Choose a task you do repeatedly (commit messages, PR descriptions, test stubs...)
  • Write a skill file in .claude/commands/your-skill-name.md
  • Include: what it does, step-by-step instructions, output format
  • Invoke it: /your-skill-name
  • Review the output — does it do what you wanted?
  • Edit the skill and invoke again (iterate at least once)
  • Commit: git add .claude/commands/ && git commit -m "feat: add [skill-name] skill"
Goal: One working, committed skill. Quality over quantity — make one skill excellent.
